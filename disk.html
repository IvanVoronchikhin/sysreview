HDD vs SSD

Два типа хранилищ с принципиально разной физической природой

HDD
Hard Disk Drive · Магнитный диск
Скорость
~180 МБ/с
IOPS
~200
Надёжность

Цена/ТБ
~20$/ТБ

Принцип: вращающиеся магнитные пластины + считывающая головка. Механика обусловливает задержки (~10 мс), шум и уязвимость к ударам. Преимущество — низкая стоимость гигабайта для больших объёмов.

VS
SSD
Solid State Drive · NAND Flash
Скорость
до 7000 МБ/с
IOPS
до 1M+
Надёжность
Высокая
Цена/ТБ
~80$/ТБ
Шум
Нет

Принцип: ячейки NAND Flash без движущихся частей. Минимальные задержки (<0.1 мс), бесшумность, устойчивость к вибрации. NVMe-версии в 10–40× быстрее SATA-SSD за счёт прямого подключения к шине PCIe.

Типы SSD по интерфейсу

SATA SSD
до 550 МБ/с

2.5" форм-фактор, разъём SATA. Замена HDD без смены интерфейса.

NVMe M.2
до 3500 МБ/с

PCIe 3.0×4, M.2 слот. Стандарт для современных ноутбуков и ПК.

NVMe PCIe 4/5
до 14 000 МБ/с

PCIe 4.0/5.0, флагманские накопители. Максимальная производительность.

Таблицы разделов: MBR vs GPT

Два стандарта разметки дисков с разными ограничениями и возможностями

Скриншот · Disk Management / diskpart — структура разделов дата/время снятия скриншота
⚠ Устаревший

MBR

Master Boot Record, 1983. Первые 512 байт диска содержат загрузчик и таблицу разделов.

Макс. размер диска2 ТБ
Первичных разделов4 (или 3+1 расш.)
СовместимостьWindows XP+, старые ПК
ЗагрузкаBIOS Legacy
Резервная копияНет
CRC-защитаНет
Плюсы
  • Широкая совместимость со старым железом
  • Поддержка BIOS-загрузки
Минусы
  • Лимит 2 ТБ
  • Только 4 первичных раздела
  • Нет защиты таблицы разделов
✓ Современный стандарт

GPT

GUID Partition Table, 2000. Часть стандарта UEFI, пришедшая на замену MBR.

Макс. размер диска9.4 ЗБ (зеттабайт)
Разделов128 (по умолчанию)
СовместимостьWindows 8+, UEFI-системы
ЗагрузкаUEFI (Secure Boot)
Резервная копияЕсть (конец диска)
CRC-защитаCRC32 заголовка
Плюсы
  • Практически неограниченный объём
  • До 128 разделов
  • Резервная копия таблицы
  • CRC-проверка целостности
Минусы
  • Требует UEFI (не работает с BIOS)
  • 32-бит Windows не загружается с GPT

Рекомендации по выбору

Практическое руководство: что и когда использовать

🖥 Системный диск ПК
  • ✓NVMe SSD PCIe 4.0 — максимальная скорость
  • ✓GPT + UEFI Secure Boot
  • ✓NTFS — единственный вариант для Windows
  • ✓Размер кластера 4096 байт (стандарт)
💾 Архивный диск / NAS
  • ✓HDD 3.5" 7200 RPM — лучшая цена/ТБ
  • ✓GPT (для дисков >2 ТБ)
  • ✓NTFS для Windows-only среды
  • ✓Крупный кластер (32–64 КБ) для больших файлов
📱 USB-флешка / SD-карта
  • ✓exFAT — если файлы >4 ГБ
  • ✓FAT32 — для максимальной совместимости
  • !NTFS — только если только Windows
🎮 Игровой диск
  • ✓NVMe SSD — DirectStorage требует NVMe
  • ✓GPT + NTFS
  • ✓Кластер 4096 — оптимален для игр

Вывод

Итог по дискам и файловым системам

Современный подход к хранению данных предполагает связку: NVMe SSD + GPT + NTFS для системного раздела и HDD + GPT + NTFS/exFAT для архивных данных. Понимание структуры MFT, журналирования и таблиц разделов необходимо для грамотной диагностики и работы с утилитами следующего раздела.

Ключевые выводы
  • NVMe SSD + GPT — стандарт 2024
  • NTFS незаменима для системного раздела Windows
  • exFAT вытесняет FAT32 для переносных устройств
  • GPT превосходит MBR по всем параметрам
Ограничения
  • FAT32: лимит 4 ГБ на файл — критично для ISO
  • HDD: высокая латентность при случайном доступе
  • NTFS: ограниченная поддержка вне Windows
